Preferences

Message rendering
How email bodies are displayed
Blocks renders structured content only. Eliminates tracking pixels, remote image loads, and CSS fingerprinting. Recommended.
AI metadata language
Language for locally-generated summaries
AI metadata is always generated locally on your device — never processed server-side by default.

Read receipts

Send read receipts
Senders who request a receipt will see when you open their message. Off by default.
In UCP, read receipts are always opt-in and declared in your identity record. Senders cannot request a receipt unless you've enabled this — the protocol makes this structurally explicit.

External images

Auto-load external images
External image URLs can be used to track when and where you opened a message. Off by default.
Always allow from Relay contacts
Load images sent by contacts on Relay (E2E only)
In Blocks rendering mode, external images are never auto-loaded. This applies to HTML messages via the email bridge only.
